# Project Retrospective

## 專案名稱

Beginner Todo App

## 專案定位

這是一個新手前端 side project。專案範圍不大，重點是把一個完整的小工具做出來：可以輸入、可以操作、可以保存資料，也有基本的手機版樣式。

## 為什麼值得做

Todo App 看起來簡單，但它很適合拿來練習資料流。使用者每做一個操作，資料會改變，畫面也要跟著改變。

如果這件事理解清楚，之後學 React 或 Vue 時，會更容易理解 state 和 render 的概念。

## 這次用了哪些技術

- 原生 HTML：建立表單、按鈕、清單與基本結構。
- 原生 CSS：完成版面、手機版排版、按鈕狀態與完成樣式。
- 原生 JavaScript：使用 DOM API、事件監聽、陣列方法與函式拆分。
- localStorage：讓資料保存在瀏覽器，重新整理後不會消失。
- GitHub：把專案保存到遠端，方便之後繼續修改。

## 開發過程中的取捨

### 沒有使用框架

這次先不用 React 或 Vue。原因是我想先熟悉原生 DOM 和事件處理，知道畫面到底是怎麼被更新的。

### 沒有使用後端

資料只存在瀏覽器的 `localStorage`。目前這樣就夠用，也讓專案保持簡單。

### 沒有做太複雜的設計

畫面先以清楚、可操作為主。等功能更完整後，再慢慢調整視覺細節。

## 遇到的困難

### 1. 狀態同步

如果勾選完成只改 checkbox，卻沒有更新 `todos` 陣列，下一次 render 就會回到舊狀態。

解法是讓所有操作都先更新資料，再呼叫 `render()`。

### 2. 空狀態

一開始沒有任務時，畫面如果完全空白會有點不清楚。所以我加了 empty state，讓使用者知道現在只是沒有資料。

### 3. id 產生方式

`crypto.randomUUID()` 很方便，但有些環境可能不支援，所以補了一個 fallback id。

## 學到的事
