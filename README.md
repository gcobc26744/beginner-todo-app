# Beginner Todo App

一個用 HTML、CSS、JavaScript 做的待辦清單 side project。這個專案的目標很單純：用最少的工具，把前端互動的基本流程走過一次。

GitHub repo: https://github.com/gcobc26744/beginner-todo-app

## 為什麼做這個專案

待辦清單很常見，但它剛好可以練到幾個前端基本功：

- 使用者輸入資料
- 程式檢查輸入內容
- 畫面根據資料更新
- 使用者可以切換完成狀態
- 資料可以保存在瀏覽器裡

我想先做一個不靠框架、不需要安裝套件的小工具，確認自己理解 DOM、事件監聽和資料更新的流程。這個專案也方便之後回來改功能，例如編輯待辦、加上日期，或改成 React 版本。

## 功能

- 新增待辦事項
- 勾選完成或取消完成
- 刪除單一待辦事項
- 篩選全部、未完成、已完成
- 顯示剩餘未完成數量
- 使用 `localStorage` 保存資料
- 支援手機和桌面版版面

## 使用技術

- HTML5：頁面結構與表單
- CSS3：版面、按鈕狀態、完成狀態、手機版樣式
- JavaScript：DOM 操作、事件監聽、陣列資料處理
- Web Storage API：用 `localStorage` 保存待辦資料
- Git / GitHub：版本控制與專案保存

## 如何使用

1. 下載或 clone 這個專案。
2. 用瀏覽器打開 `index.html`。
3. 在輸入框輸入待辦事項，按下 `Add`。

不需要安裝套件，也不需要啟動伺服器。

## 開發時碰到的問題

### 1. 畫面要跟資料同步

新增、刪除、勾選完成後，不能只改畫面。資料也要一起更新，不然篩選或重新整理時就會出現不一致。

這個專案把 `todos` 陣列當作主要資料。每次操作都先更新 `todos`，再呼叫 `render()` 重新畫畫面。

### 2. 直接開 HTML 也要能用

我希望這個專案對新手友善，所以沒有加入 build tool 或套件管理。只要打開 `index.html` 就能跑。

### 3. 空清單也要有回饋

如果清單裡沒有任務，畫面不能只是一片空白。所以我加了 empty state，讓使用者知道目前沒有資料。

## 學到的事

- 前端互動常常可以拆成「更新資料」和「更新畫面」。
- README 要寫得讓未來的自己也看得懂。
- `localStorage` 很適合這種只存在本機的小工具。
- 篩選功能不需要建立多份清單，只要從同一份資料算出要顯示的內容。
- 小專案也可以留下開發筆記，之後回來看會比較知道當時怎麼想。
