# Beginner Todo App

一個用 HTML、CSS、JavaScript 做的待辦清單 side project。這不是只為了做出功能，而是用一個小專案練習前端最基本、也最常在面試被問到的能力：畫面結構、使用者互動、狀態管理、資料保存與專案說明。

GitHub repo: https://github.com/gcobc26744/beginner-todo-app

## 為什麼做這個專案

待辦清單是很經典的新手專案，但它不只是「新增一行文字」而已。它可以練到一個前端產品最小但完整的流程：

- 使用者輸入資料
- 程式驗證輸入內容
- 畫面根據資料重新渲染
- 使用者可以修改狀態
- 資料可以保存在瀏覽器裡

這個專案適合當成第一個放上 GitHub 的作品，因為它小、好理解，也很容易延伸。之後面試時可以用它說明自己如何拆解需求、如何設計資料結構、如何讓畫面和資料保持同步。

## 專案功能

- 新增待辦事項
- 勾選完成或取消完成
- 刪除單一待辦事項
- 篩選全部、未完成、已完成
- 顯示剩餘未完成數量
- 使用 `localStorage` 保存資料
- 支援手機和桌面版版面

## 使用技術

- HTML5：建立語意化頁面結構
- CSS3：製作響應式版面、按鈕狀態、完成狀態樣式
- JavaScript：處理事件、資料陣列、畫面渲染
- Web Storage API：使用 `localStorage` 保存待辦資料
- Git / GitHub：版本控制與專案發佈

## 如何使用

1. 下載或 clone 這個專案。
2. 用瀏覽器打開 `index.html`。
3. 在輸入框輸入待辦事項，按下 `Add`。

不需要安裝套件，也不需要啟動伺服器。

## 開發時碰到的問題

### 1. 畫面要跟資料同步

新增、刪除、勾選完成之後，如果只改畫面，不改資料，下一次篩選或重新整理就會出錯。所以這個專案把 `todos` 陣列當成主要資料來源，每次資料變動後都呼叫 `render()` 重新畫畫面。

### 2. 直接開 HTML 也要能用

新手專案最好不要一開始就要求安裝很多工具，所以這個專案避免使用需要 build 的框架。JavaScript 的 id 產生也加了 fallback，讓瀏覽器支援度更穩。

### 3. 空資料狀態也要處理

如果清單是空的，畫面不能只剩一塊空白。因此加了 empty state，讓使用者知道現在沒有任務，也知道下一步可以新增。

## 學到了什麼

- 前端互動的核心是「資料改變後重新渲染畫面」
- 小專案也要有清楚的資料夾結構與 README
- `localStorage` 適合保存簡單、少量、只在本機使用的資料
- 篩選功能可以用同一份資料衍生不同畫面，不需要建立多份清單
- 做 side project 時，寫下開發原因與心得比只放程式碼更有價值

## 面試可以怎麼介紹

可以這樣說：

> 這是一個用原生 HTML、CSS、JavaScript 做的待辦清單。我用它練習前端最基本的狀態管理，包含新增、刪除、完成狀態、篩選和 localStorage 保存資料。開發時我把資料集中在 `todos` 陣列，任何操作都先更新資料，再重新 render 畫面，讓 UI 和資料保持一致。

如果被問「為什麼不用框架」，可以回答：

> 因為這是新手基本功專案，我想先理解 DOM 操作、事件監聽和狀態更新的流程。等這些概念熟了，再用 React 或 Vue 會更清楚框架幫我解決了什麼問題。
